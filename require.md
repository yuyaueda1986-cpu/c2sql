
# プロジェクト「c2sql」の概要

## 1. 開発の背景と解決する課題
C言語からRDBを操作する場合、SQL文の動的生成、型のバインディング、メモリへのマッピングなど、非常に煩雑な手続きが必要です。「c2sql」はこれらの面倒な手続きを内部で完全に隠蔽し、C言語の「構造体」とDBの「テーブル」をシームレスに結びつけます。

## 2. 最大の特徴：構造体の自動マッピング
本ライブラリの最大の特徴は、初期化時に構造体のレイアウト（メンバの型や位置）を一度登録するだけで、以降は構造体名とポインタを渡すだけでDBへの保存・取得が完了する点です。ライブラリ内部で適切なSQL（INSERT, SELECT, UPDATEなど）を自動生成し、実行します。

## 3. APIの利用イメージと仕組み
利用者は以下のような3ステップで、直感的にデータベースを操作できます。

### ステップ1：DBの初期化と、構造体情報（スキーマ）の登録
最初に、C言語の構造体がDB上でどのようなテーブル・カラム構成になるかをライブラリに教える「セットアップ処理」を行います。（内部的には offsetof マクロなどを用いてメモリ配置を解析・登録します）。

```
// ① データベースの初期化
DB識別子 = SqlRDBInit("接続先DB名");

// ② 構造体のメンバ情報（型、オフセット位置、サイズなど）を定義
// 例：UserData構造体（id, name, age）の情報を配列などで定義
SqlRDBColumnDef cols[] = {
    {"id",   SQL_TYPE_INT,  offsetof(UserData, id),   sizeof(int)},
    {"name", SQL_TYPE_TEXT, offsetof(UserData, name), sizeof(char)*32},
    {"age",  SQL_TYPE_INT,  offsetof(UserData, age),  sizeof(int)}
};

// ③ 構造体名（テーブル名）と紐付けてライブラリに登録
SqlRDBRegisterStruct(DB識別子, "UserData", cols, カラム数);
```

### ステップ2：データの書き込み（保存）
登録が完了すれば、あとは「構造体名」と「ポインタ」を渡すだけです。ライブラリが自動でテーブルの有無を確認し、SQLを構築してデータを挿入します。

```
UserData u = {1, "山田太郎", 30};

// "UserData"テーブルに、構造体の中身を保存
SqlRDBWrite(DB識別子, "UserData", &u);
```

### ステップ3：データの読み込み（取得）
読み込み時も同様に、取得したい条件（キーなど）と、データを受け取るための構造体ポインタを渡すだけで、ライブラリがDBのレコードをC言語の構造体に復元します。

```
UserData u;

// "UserData"テーブルからデータを取り出し、構造体 u にセット
SqlRDBRead(DB識別子, "UserData", 検索条件, &u);
```

## まとめ
「c2sql」は、初期セットアップで「C言語の構造体」と「DBのテーブル」のマッピングを定義することで、以降のRDBアクセスを劇的に簡略化します。開発者はSQLの構築や型変換のバグに悩まされることなく、安全かつ高速にデータ永続化処理を実装できるようになります。
